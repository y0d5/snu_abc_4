슬라이드 4
  • AI 시대에는 아이디어가 있으면 실제로 앱을 만들 수 있는 진정한 전문가 시대가 열렸다
  • 미켈란젤로의 조각 철학처럼 LLM에는 모든 답이 있으며, 이를 얼마나 정확히 끄집어내느냐가 관건이다
  • 기존 프로그래밍 언어에 의존하던 엔지니어와 달리, 특정 도메인 전문 지식을 가진 사람이 AI를 통해 답을 끄집어낼 수 있다
  • 핵심 경쟁력은 프롬프트를 정확히 구사하고 AI Agent를 정확히 정의하는 능력에 달려 있다
  • HITL(Human In The Loop)에서 사람이 개입할 지점을 적절히 설계하는 것이 중요하다
슬라이드 5
  • AI 시대에는 개발 비용이 획기적으로 감소하여 작은 규모의 앱들이 폭발적으로 등장할 것이다
  • 과거에는 개발 비용과 시간이 많이 소요되어 최대한 많은 사용자를 대상으로 하는 앱만 개발했다
  • 현재는 개인의 특정 필요에 의한 소규모 앱도 쉽게 개발할 수 있으며, 이를 범용 앱으로 확장할 수 있다
  • 실제 사례로 서류 작성 자동화 앱을 20-30분 만에 개발하여 업무 효율성을 크게 향상시켰다
  • 개발자가 글을 쓰듯이 앱을 만들 수 있는 환경이 조성되어 개인 맞춤형 솔루션의 개발이 일상화되고 있다
슬라이드 6
  • 개인의 특정한 니즈를 해결하는 작은 AI 앱들이 폭발적으로 증가할 것이다
  • 영수증 사진으로 자동 정산하는 앱 사례처럼, 누구든지 필요한 앱을 직접 만들어 사용하는 시대가 온다
  • 개인이 만든 작은 AI Agent들이 거래되는 거대한 인력 시장이 인터넷에 형성될 것이다
  • 여러 AI Agent들을 결합하여 더 복합적이고 상위 수준의 업무를 처리하는 구조로 발전한다
  • Agent to Agent(A2A) 프로토콜을 통해 AI Agent들을 묶어 활용하는 생태계가 구축된다
슬라이드 7
  • AI Agent시대에는 전문가가 Agent의 Persona(Role, Goal, Backstory)와 Task(Description, Expected Output, Instructions)를 정의하는 역할을 담당한다
  • AI Agent의 성능을 좌우하는 결정적 요소는 프로그래밍 언어가 아닌 자연어로 작성된 명확한 프롬프트이다
  • 도메인 전문가만이 할 수 있는 전문적 지식을 자연어로 정의하여 AI Agent에게 부여할 수 있다
  • 전문적 지식을 담은 AI Agent 앱들이 대량으로 등장하여 업무 효율성이 크게 향상될 것이다
  • 레이 커즈와일이 언급한 바와 같이 Vibe Coding은 인간의 엄청난 잠재력을 폭발시킬 것이다
슬라이드 8
  • AI Agent는 Role(역할), Goal(목표), Backstory(이력서)를 정의하여 인터넷 상의 전문 작업자를 만드는 개념이다
  • 유튜브 시나리오 작성 시스템은 Researcher(조사), Creator(기획), Writer(작성) 3개 에이전트가 협업하여 작동한다
  • 에이전트 간 데이터 전달은 기존 프로그래밍의 변수 연결 없이 자연어로 '리서치 결과를 바탕으로'라고 명시하면 자동으로 처리된다
  • Backstory에서 '15년 차 전문가', '100만 뷰 달성 경험' 등 원하는 전문성을 기술하면 해당 수준으로 작업을 수행한다
  • AI Agent 시대에는 특정 영역의 전문 지식과 정확한 프롬프트 정의 능력을 가진 전문가가 핵심 역할을 하게 된다
슬라이드 9
  • Prompt Engineering에서 Context Engineering으로 발전해야 하며, 단순한 지시를 넘어 근거 자료를 함께 제공하는 것이 중요하다
  • Context는 AI가 작업할 때 참고할 수 있는 구체적인 자료나 데이터를 의미한다
  • 상품 소개 이메일 작성 시 상대방의 이메일 본문과 상품 소개서를 Context로 제공하면 더 정확하고 맞춤형 결과를 얻을 수 있다
  • Context Engineering은 향후 RAG(Retrieval Augmented Generation), Vector DB, AI Agent 연동 등 고급 AI 활용의 기초가 된다
  • 어떤 자료를 Context로 제공할지 정확히 판단하는 것이 효과적인 AI 활용의 핵심이다
슬라이드 10
  • Context Engineering에서는 올바른 정보를 올바른 시간에 올바른 형식으로 제공해야 한다
  • 올바른 정보로는 기반 컨텍스트(이메일, 소개서), 영구적 규칙(.cursorrules), 구체적 작업계획(PRD) 등이 있다
  • 올바른 시간에 제공한다는 것은 현재 프로세스에서 꼭 필요한 것만 골라서 정확한 타이밍에 넣어주는 것이다
  • Context Window Size와 토큰 사용량을 고려하여 무작정 모든 정보를 넣지 말아야 한다
  • 올바른 형식으로는 JSON 포맷을 사용해야 하며, 이는 AI가 가장 정확히 이해하는 데이터 포맷이다
슬라이드 11
  • Cursor AI 코딩에서 애매한 설명 대신 전문 용어인 'Context Word'를 사용하면 정확하고 빠른 결과를 얻을 수 있다
  • CRUD(Create, Read, Update, Delete)를 사용하면 항목의 생성, 읽기, 수정, 삭제 기능을 한 번에 요청할 수 있다
  • bfcache(Back-Forward Cache)를 적용하면 뒤로 가기 시 페이지 재로딩 없이 이전 페이지를 즉시 표시할 수 있다
  • Lighthouse Audit를 실행하면 웹 앱의 성능 최적화를 자동으로 분석하고 적용해준다
  • vibecodic.com 사이트에서 Vibe Coding에 필요한 50개의 Context Word를 학습할 수 있다
슬라이드 12
  • 모달리티(Modality)는 원래 행태학 용어로 사람의 오감을 의미하며, AI에서는 AI가 처리하는 데이터의 형태나 종류를 뜻한다
  • AI가 처리하는 주요 모달리티에는 텍스트, 이미지, 음성, 음악, 비디오, 코드뿐만 아니라 분자 구조, 로봇 동작까지 포함된다
  • LLM이 분자 구조를 처리함으로써 바이오 기술 발달이 가속화되고, 로봇 동작을 생성하여 피지컬 로봇 기술이 발전하고 있다
  • AI가 어떤 모달리티를 어떤 방식으로 처리하는지 이해해야 적절한 기능 선택과 앱 개발 방향을 설정할 수 있다
  • 각 모달리티의 특징을 알고 있으면 AI에게 컨텍스트를 효과적으로 제공할 수 있다
슬라이드 13
  • 텍스트 파일 형식에 따라 AI의 출력 품질이 크게 달라지므로 목적에 맞는 형식 선택이 중요하다
  • TXT 파일은 처리 속도가 빠르고 호환성이 좋지만 구조 체계나 표, 그래프 처리에 한계가 있다
  • Markdown은 문단 구조와 강조 표시가 가능하며 AI가 가장 잘 이해하는 형식으로 효율적이다
  • HTML/XML은 태그를 통해 내용의 의미를 명시적으로 구분할 수 있어 AI가 정확히 인식한다
  • PDF는 AI가 광학 처리를 통해 추정으로 단락을 나누므로 정확도가 떨어져 MS Word 파일이 더 나은 선택이다

슬라이드 15
  • AI는 이미지를 단순히 보는 것이 아니라 내용을 읽어내며, 이는 모달리티 처리 과정을 통해 이루어진다
  • AI의 모달리티 처리는 표현(Representation), 정렬(Alignment), 추론(Reasoning), 생성(Generation), 전이(Transference)의 5단계로 진행된다
  • 표현 단계에서는 픽셀 데이터에서 선, 모양, 질감, 객체 등 의미있는 시각적 요소를 식별한다
  • 정렬 단계에서는 식별된 특징들을 수천 차원의 잠재공간(Latent Space)에 벡터로 변환하여 배치하며, 현재는 텍스트, 이미지, 음성 등 모든 모달리티가 하나의 공간에 통합되어 있다
  • AI는 모든 정보를 임베딩을 통해 숫자로 변환하여 처리하므로, 텍스트, 이미지, 음성 등의 구분 없이 동일한 방식으로 처리한다

슬라이드 20
  • 인터넷의 국제화를 위해 Unicode(1987), ICU(1999), CLDR(2003) 등 다양한 표준화 시도가 있었음
  • AI 시대에 이러한 국제화 기술들이 i18n(Internationalization)으로 통합됨
  • i18n은 앱 개발 초기에 적용하면 다국어 버전을 동시에 자동 생성하는 기능을 제공함
  • i18n 적용 시 원본 언어 콘텐츠를 수정하면 다른 언어 버전도 자동으로 업데이트됨
  • 사용자의 브라우저 언어 설정에 따라 해당 언어로 자동 표시되는 기능을 구현할 수 있음
슬라이드 21
  • i18n은 고정 콘텐츠 위치를 key로 지정하고, 해당 key에 여러 언어의 value를 JSON 형식으로 저장하여 다국어를 지원한다
  • 사용자의 기기나 브라우저 설정 언어에 따라 자동으로 해당 언어로 콘텐츠가 표시되며, 수동 변경 없이도 작동한다
  • 앱 개발 초기부터 i18n을 적용해야 하며, 나중에 추가하는 것과 처음부터 적용하는 것의 차이가 매우 크다
  • 한국은 국내 서비스 먼저 개발 후 글로벌 확장하는 단계적 접근을, 미국은 처음부터 글로벌 시장을 대상으로 하는 Born-Global 접근을 한다
  • i18n 구현 시 페이지에는 key 값만 삽입하고, 실제 텍스트는 언어별 JSON 파일에서 가져와 표시하는 방식으로 작동한다
슬라이드 22
  • 앱 개발 초기부터 i18n(국제화)을 디폴트로 적용해야 하며, 나중에 추가하면 복잡해진다
  • LLM은 언어별로 차등화된 성능을 제공하며, top-tier 7개 언어는 네이티브 수준, second-tier(한국어 포함)는 비즈니스 활용 가능 수준이다
  • 50-70개 언어에서 일상적인 질의응답 및 번역이 가능한 수준까지 지원된다
  • 언어 장벽이 무너진 환경에서는 로컬 시장 우선 접근 방식을 버리고 처음부터 글로벌 시장을 타겟해야 한다
  • 과거와 달리 현재는 메뉴와 고정 콘텐츠뿐만 아니라 전체적인 다국어 지원이 LLM을 통해 가능하다
슬라이드 23
  • 기존에는 고정 콘텐츠만 다국어화가 가능했지만, 현재는 LLM을 통해 동적으로 생성되는 콘텐츠도 실시간 다국어 처리가 가능하다
  • AI 에이전트를 활용하여 언어 감지, 감성 분석, 다국어 생성을 자동화할 수 있다
  • 이메일 상담 시스템에서 4개의 에이전트가 동시에 작동하여 언어 감지 → 감성 분석 → 답변 생성 → 후속 처리 여부 판단을 자동화한다
  • 어떤 언어로 문의가 와도 해당 언어로 자동 답변하고, 담당자에게는 한국어로 보고하는 시스템 구축이 가능하다
  • 처음부터 글로벌 서비스를 염두에 두고 시스템을 설계할 수 있는 환경이 조성되었다
슬라이드 24
  • AI 도구들이 앱 테스트 전 과정(테스트 데이터 생성, 작동 검증, 사용자 테스트)을 자동화하여 전문가 없이도 완벽한 테스트가 가능하다
  • ACCELQ, TestRigor 등 AI 기반 테스트 자동화 플랫폼은 웹사이트 주소만으로 자동 테스트 및 상세 보고서를 생성한다
  • CWE, NVD 등 국제 보안 표준을 Context Word로 활용하면 전문 지식 없이도 앱에 표준 수준의 보안 조치를 자동 적용할 수 있다
  • SEO 최적화, 접근성(WCAG) 등 과거 전문 에이전시가 담당했던 업무들이 적절한 Context Word 사용으로 자동화 가능하다
  • 정확한 표준 용어(Context Word)를 알면 복잡한 전문 업무들을 AI에게 정확하게 지시할 수 있어 전문가 의존도가 크게 감소한다
