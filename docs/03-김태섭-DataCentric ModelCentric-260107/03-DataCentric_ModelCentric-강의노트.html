슬라이드 1

슬라이드 2
  • 강연자는 서울대학교 데이터 사이언스 대학원 교수로 5년째 재직 중
  • 산업 경험을 바탕으로 늦게 박사 과정을 시작하여 학계로 전환
  • 아마존과 카카오 브레인에서 인공지능 분야 실무 경험을 쌓음
슬라이드 3
  • 머신러닝은 인간이 과거 경험으로 학습하는 것과 달리 데이터와 지시사항으로 학습한다
  • 머신러닝의 기본 구조는 입력 데이터 X와 목표 데이터 Y, 그리고 예측 함수로 구성된다
  • LAAL 연구실은 Learning and Adaptation Algorithm을 연구하며 25-30명 규모로 운영된다
  • 표현 학습(Representation Learning)은 현재 파운데이션 모델과 거대 언어 모델의 핵심 기술이다
슬라이드 4
  • 파운데이션 모델(기본이 되는 AI 모델)을 만드는 기술을 연구하고 있다
  • 트랜스퍼 러닝을 통해 파운데이션 모델을 실용적으로 산업에 적용하는 방법을 연구한다
  • 파운데이션 모델의 제작과 활용이라는 큰 틀에서 연구를 진행하고 있다
  • 산업 현장에서 AI 기술 적용 시 발생하는 현실적 문제들에 집중하여 연구하고 있다
슬라이드 5
  • 피지컬 AI는 파운데이션 모델 구축, 활용, 현실 문제 적용을 포함하는 종합적인 AI 기술이다
  • 기존 GPT와 달리 피지컬 AI는 컴퓨터나 앱 환경을 벗어나 물리 세계에서 작동한다
  • 피지컬 AI는 복잡한 물리적 환경을 경험하고 상호작용해야 하는 더 도전적인 문제를 다룬다
슬라이드 6
  • 머신러닝은 크게 두 단계로 구성된다: (1) 특성 공학(Feature Engineering) - 고차원 입력 데이터를 저차원 특성 데이터로 변환, (2) 모델 피팅(Model Fitting) - 특성 데이터를 이용한 예측 모델 학습
  • AI 발전 접근법은 모델 센트릭 AI와 데이터 센트릭 AI로 구분된다
  • 데이터 센트릭 AI는 최근 5년간 주목받은 키워드이다
  • 그 이전까지는 주로 모델 센트릭 AI 컨셉으로 인공지능이 발전했다
슬라이드 7
  • 머신러닝은 사람의 학습 과정을 기계적으로 구현하려는 접근법이다
  • 사람은 과거의 경험을 통해 지식을 습득하고 학습한다
  • 기계는 직접적인 경험 대신 데이터를 통해 간접적으로 경험을 학습한다
  • 피지컬 AI의 경우 로봇이 환경과의 상호작용을 통해 직접적인 경험을 할 수 있다
슬라이드 8
  • 머신러닝은 기계에게 데이터와 활용 방법에 대한 인스트럭션을 주어 학습시키는 개념이다
  • 머신러닝은 경험 기반의 데이터를 기계에게 학습시키는 과정이다
  • 데이터는 입력값(input)과 그에 대응되는 출력값(output)의 쌍으로 구성된다
  • 머신러닝의 학습 과정은 수학적 관점에서 함수를 학습하는 것으로 이해할 수 있다
슬라이드 9
  • 머신러닝에서 함수는 입력값을 받아 출력을 생성하여 예측이나 판단을 수행하는 주체이다
  • 머신러닝은 Feature Engineering(특징 엔지니어링) 개념에서 시작된 오래된 학문이다
  • Feature Engineering은 Feature Selection(특징 선택)과 Feature Extraction(특징 추출)으로 구분된다
  • Feature Selection은 데이터에서 중요한 특징을 선택하는 과정이다
  • Feature Extraction은 주어진 데이터에서 새로운 특징을 생성하여 고차원 데이터를 저차원으로 변환하는 과정이다
슬라이드 10
  • 고차원 데이터의 예로 핸드폰으로 찍는 이미지는 픽셀 수가 많아 엄청난 고차원 데이터에 해당한다
  • 피처 익스트랙터는 고차원 데이터를 저차원 형태로 효과적으로 압축하는 기술이다
  • 머신러닝에서 복잡한 데이터를 효율적으로 다룰 수 있는 간단한 형태로 변환하는 것이 핵심 기술 중 하나이다
  • Feature Engineering 과정에서 고차원 입력 데이터를 저차원 특징 데이터로 변환한 후 모델 학습을 진행한다
슬라이드 11
  • 피처 엔지니어링은 고차원 데이터에서 저차원의 의미 있는 특성을 추출하는 과정이다
  • 피처 엔지니어링은 복잡한 데이터에서 중요한 정보만을 선별하여 불필요한 정보를 제거하는 과정이다
  • 전통적으로 피처 엔지니어링은 해당 도메인에 대한 전문 지식을 가진 사람들이 수행했다
  • 머신러닝의 예측 과정은 데이터를 구분하는 경계선(함수)을 찾는 모델 피팅 과정이다
  • 데이터의 차원이 높아질수록(만 차원, 10만 차원) 적절한 구분 함수를 찾는 것이 매우 어려워진다
슬라이드 12
  • 머신러닝은 피처 엔지니어링과 모델 피팅 두 단계로 구성된다
  • 피처 엔지니어링은 고차원 데이터를 저차원의 유의미한 특징으로 변환하는 과정이다
  • 데이터 분석가의 수작업을 자동화한 형태가 피처 엔지니어링이다
  • 모델 피팅은 정리된 데이터로 실제 예측 함수를 학습하는 과정이다
  • 머신러닝은 수학과 최적화 이론을 기반으로 한 학문이다
슬라이드 13
  • 머신러닝은 크게 피처 엔지니어링(Feature Engineering)과 모델 피팅(Model Fitting) 두 가지 요소로 구성된다
  • 피처 엔지니어링이 머신러닝에서 가장 핵심적이고 어려운 부분이다
  • 고차원 데이터를 잘 정리해야만 학습 과정이 쉬워진다
  • 피처 엔지니어링에는 도메인 지식이 필수적이며 사람의 손이 많이 가는 작업이다
  • 실제 얼굴 인식 기술 개발에서도 효과적인 특징을 찾는 과정이 경험적으로 이루어진다
슬라이드 14
  • 전통적인 머신러닝에서는 경험적으로만 데이터를 선택하다 보니 설명하기 어려운 문제들이 많이 발생했다
  • 데이터량이 급격히 증가하면서 사람이 모든 데이터를 일일이 분석하고 인사이트를 찾기가 어려워졌다
  • 기존에는 피처 엔지니어링을 사람이 수행할 수 있다고 가정했지만, 데이터량 증가와 문제 복잡도 상승으로 인해 사람의 기여도가 제한적이 되었다
  • 전통적인 머신러닝은 데이터량 증가에 따른 확장성(scalability) 문제를 가지고 있다
슬라이드 15
  • 전통적인 머신러닝은 피처 엔지니어링을 기반으로 한 모델 피팅 패러다임을 사용했다
  • 데이터가 복잡해질수록 사람의 개입을 통한 피처 엔지니어링이 어려워지고 비용이 증가한다
  • 딥러닝은 고차원 입력 데이터로부터 피처 추출기와 예측 모델을 함께 학습하는 end-to-end 학습 방식이다
  • 전통적인 머신러닝은 데이터양이 일정 수준 이상 증가하면 더 이상 성능이 향상되지 않는 한계가 있다
슬라이드 16
  • 전통적 머신러닝은 데이터가 많아도 제한된 특성 추출 능력으로 인해 활용할 수 없는 한계가 있었다
  • 데이터가 적은 상황에서는 어중간한 수준의 지능만 가질 수 있는 것이 기존 머신러닝의 한계점이었다
  • 이러한 한계점은 2000년대 초반 머신러닝 발전을 저해하는 크리티컬한 문제점으로 대두되었다
  • 딥러닝은 이러한 전통적 머신러닝의 한계를 해결하기 위해 등장한 기술이다
  • 딥러닝은 머신러닝의 특정 세부 분야로 이해해야 한다
슬라이드 17
  • 딥러닝은 기존의 피처 익스트랙터와 모델을 분리하는 방식의 문제점을 해결한다
  • 딥러닝에서는 피처 익스트랙터와 모델을 따로 보지 않고 하나의 큰 형태로 통합한다
  • 고차원 데이터를 저차원으로 변형하는 중간 단계에 인간이 개입할 필요가 없다
  • 고차원 데이터를 바로 예측 결과로 변환하는 과정을 end-to-end로 학습한다
  • 이러한 end-to-end 학습이 딥러닝의 핵심 모티베이션이다
슬라이드 18
  • End-to-end 러닝은 중간 단계에 개입하지 않고 복잡한 입력을 원하는 출력으로 직접 변환하는 방식이다
  • 피처 익스트랙터에서 사람의 개입 요소를 최소화하면서 학습을 진행한다
  • 딥러닝을 통해 데이터가 늘어날수록 성능이 지속적으로 향상될 수 있다는 기대를 갖게 되었다
  • 뉴럴 네트워크는 사칙연산의 조합으로 복잡한 함수 형태를 구현하는 획기적인 디자인이다
  • 뉴럴 네트워크의 사이즈가 커지면 함수가 복잡해지며 선형에서 비선형 구조로 발전할 수 있다
슬라이드 19
  • 딥러닝은 데이터량이 증가할수록 성능이 지속적으로 향상되는 확장성을 가지고 있다
  • 딥러닝은 뉴럴 네트워크(인공신경망) 아키텍처를 사용하여 함수를 디자인한다
  • 뉴럴 네트워크의 크기가 클수록 더 복잡한 형태의 곡선을 표현할 수 있어 함수 표현력이 향상된다
  • 뉴럴 네트워크를 통해 복잡하고 정교한 지식을 담은 함수를 학습할 수 있다
  • AI 기술 중 머신러닝이 있고, 그 중에서 뉴럴 네트워크를 사용하는 방법론이 딥러닝이다
슬라이드 20
  • AI는 학습 기반 기술과 검색 기반 기술을 모두 포함하는 개념이다
  • 구글 검색과 같은 지식 검색 기술은 학습 과정 없이도 구현 가능한 AI 기술이다
  • 최근 학습 기반 AI의 획기적 발전으로 머신러닝과 AI가 유사한 개념으로 인식되고 있다
  • 딥러닝의 뉴럴 네트워크는 복잡한 함수를 구현할 수 있다
슬라이드 21
  • 뉴럴 네트워크는 함수의 표현력을 극대화시킬 수 있는 핵심 요소이다
  • 뉴럴 네트워크(인공신경망)는 1943년에 처음 등장한 기술로, 사람의 뉴런 원리를 수학적으로 모델링한 방법이다
  • 딥러닝의 성공에는 뉴럴 네트워크 외에도 대량의 데이터가 필수적인 요소이다
  • 데이터를 많이 사용할수록 인공지능의 성능이 향상된다는 가정이 중요하다
  • 딥러닝 발전의 마지막 핵심 요소는 GPU의 활용이다
슬라이드 22
  • GPU의 급속한 발전이 현재 딥러닝 혁신의 핵심 동력이다
  • 2015년과 현재의 GPU 성능 차이가 매우 크다
  • 데이터와 뉴럴 네트워크 이론은 예전부터 존재했지만 GPU만 급속도로 발전했다
  • 뉴럴 네트워크는 70-80년 전부터 있던 기존 이론이다
  • GPU 발전이 GPT와 같은 새로운 AI 기술 발전을 가능하게 했다
슬라이드 23
  • GPU(NVIDIA)의 딥러닝 발전에서의 중요한 역할과 영향력
슬라이드 24
  • AI 버블론에도 불구하고 피지컬 AI의 등장으로 AI가 컴퓨터와 핸드폰을 벗어나 현실 세계로 확장되고 있다
  • 피지컬 AI의 발전으로 GPU에 대한 수요가 지속적으로 증가할 것으로 예상된다
  • NVIDIA는 GPU 기술 분야에서 독보적인 위치를 유지하고 있으며 경쟁사들이 따라잡기 어려운 상황이다
  • NVIDIA의 기술적 우위가 주식 가격에 반영되고 있다
슬라이드 25
  • 딥러닝에서 빅데이터와 뉴럴 네트워크뿐만 아니라 GPU의 역할이 상당히 중요했다
  • 데이터 센트릭 접근법에서 데이터의 역할이 매우 중요하다
  • 양질의 데이터가 필요하며, 필요한 정보가 포함된 데이터가 진짜 가치를 가진다
  • 필요하지 않은 정보가 들어있는 데이터는 양이 많아도 의미가 없다
  • 빅데이터는 딥러닝의 중요한 요소 중 하나이다
슬라이드 26
  • AI 기술은 1950년대부터 발전해왔으나, 딥러닝은 2010년 초반부터 실질적으로 시작되었다
  • 요슈아 벤지오(몬트리올대 교수)는 2010년대에 기존 머신러닝으로는 AI 발전이 불가능하다며 딥러닝의 필요성을 주장했다
  • 딥러닝은 AI 발전의 새로운 전환점이 되었으며, 2015년경부터 본격적인 발전이 시작되었다
슬라이드 27
  • 딥러닝의 발전으로 인해 최근 노벨상까지 수상하는 등 엄청난 발전을 이루었다
  • 딥러닝은 다량의 데이터를 활용하여 효용성을 극대화하는 기술이다

  • Smoothness 가정은 동일한 객체에 약간의 변형을 가해도 그 객체의 본질적 특성은 보존된다는 원리에 기반한다

  • 예를 들어 사람 수 세기 작업을 위해서는 실제로 사람이 카운트한 정답 데이터가 필요하다

  • 현업에서 새로운 서비스나 기능 추가 시마다 레이블 데이터를 새로 만들어야 하는 비용 문제가 발생한다
  • 정답이 없는 상황에서 학습 목적을 어떻게 디자인할 것인가가 핵심 과제다
  • 기존 지도학습은 정답과 연결된 목적함수를 사용했지만, 이제는 데이터 자체를 이해하는 방식으로 학습 목표를 재정의해야 한다
  • 레이블 없이 데이터만으로 학습 목적을 달성하는 새로운 접근 방식이 필요하다
슬라이드 63
  • 기존 딥러닝 방법론은 새로운 서비스마다 데이터 수집과 학습을 반복해야 하는 지속가능성 문제가 있다
  • Self-Supervised Learning은 레이블이 없는 데이터만으로 학습하는 새로운 패러다임이다
  • Self-Supervised Learning은 고차원 데이터를 저차원으로 변환하는 과정을 학습하여 범용적으로 활용 가능하다
  • Self-prediction 방식은 데이터의 일부분을 가리고 나머지 부분으로 예측하는 방법으로 학습한다
  • 시계열 데이터 예시에서 앞부분 데이터로 뒷부분을 예측하도록 문제를 재정의하여 학습한다
슬라이드 64
  • Self-supervised learning에서는 데이터 자체가 입력이자 출력이 되는 정답 역할을 한다
  • 시계열 관점에서 미래를 예측하는 것이 하나의 태스크가 된다
  • 비디오 데이터의 경우 과거 프레임을 주고 미래 프레임을 예측하도록 학습한다
  • 전체 데이터를 가지고 있지만 뒷부분을 모르는 것처럼 가정하여 학습을 진행한다
  • 현재 시점만 보고 미래를 예측하는 방식으로 문제를 정의할 수 있다
슬라이드 65
  • Self-Supervised Learning에서 자기 예측(self-prediction)은 생성(generation)과 동일한 패러다임이다
  • 비디오에서는 현재 프레임을 주고 미래 프레임을 생성하는 방식으로 학습한다
  • 이미지에서는 일부 영역을 마스킹하고 나머지 부분을 예측하여 생성하는 방식을 사용한다
  • 예측의 결과가 곧 데이터 생성이므로 Self-Supervised Learning이 생성형 AI로 연결된다
  • GPT의 학습 원리도 이러한 예측 기반 생성 방식을 따른다
슬라이드 66
  • Self-Supervised Learning은 인터넷의 날것 텍스트 데이터에서 일부 단어를 숨기고 그것을 맞추도록 학습하는 방식이다
  • GPT는 다음 단어를 예측하는 단순한 원리로 학습되지만 강력한 성능을 보여준다
  • Self-Supervised Learning에서는 같은 데이터가 입력이자 출력이 되는 형태로 문제를 변형한다
  • 이미지의 경우 픽셀 단위가 아닌 패치 단위로 처리하여 Self-Supervised Learning을 적용한다
슬라이드 67
  • Self-Supervised Learning은 레이블이 없는 데이터셋을 활용하는 딥러닝 방법론이다
  • 표현 학습(representation learning)을 통해 특징 추출기(feature extractor) 역할을 수행한다
  • 데이터의 속성을 이해하고 파악하는 것이 목적이다
  • 자기 예측(self-prediction) 기반의 학습 방식을 사용한다
슬라이드 68
  • Self-Supervised Learning에서 이미지를 패치 단위로 분할하여 학습 데이터를 생성한다
  • 일부 패치를 검은색으로 마스킹하여 모르는 부분으로 설정하고, 나머지 부분을 힌트로 제공한다
  • 모델은 마스킹된 부분을 예측하여 원본 이미지를 복원하는 방식으로 학습한다
  • 이 과정을 통해 모델이 데이터 생성 과정에 대한 근본적인 이해를 획득한다
  • 사람이 퍼즐을 맞추거나 세상을 이해하는 방식과 유사한 학습 메커니즘이다
슬라이드 69
  • 퍼즐 조각을 맞춰나가는 과정이 현재 생성형 AI의 생성 과정과 유사하다
  • 이러한 학습 방식을 통해 생성형 AI 패러다임이 크게 발전했다
  • GPT와 같은 언어 모델은 다음 단어를 예측하는 방식으로 학습하고 작동한다
  • 이러한 AI의 작동 방식이 진정한 지능인지에 대한 근본적 질문이 제기된다
  • 현재 AI의 접근 방식은 본질적으로 통계적 접근법이다
슬라이드 70
  • Self-Supervised Learning은 unlabeled dataset을 활용하여 다음에 올 데이터를 확률적으로 예측하는 방식으로 학습한다
  • 인간의 사고 과정은 항상 언어 기반으로 동작하며, 이는 사고의 가장 기본적인 메커니즘이다
  • GPT나 Gemini 같은 언어모델들이 간단한 원리로도 뛰어난 성능을 보이고 있다
  • 현재 AI 모델들은 인간이 가진 메타인지(metacognition) 능력이 부족하다는 한계가 있다

  • 이러한 인간의 자연스러운 학습 과정이 Self-Supervised Learning의 기본 원리로 볼 수 있다

📌 Key Takeaways

  • AI 발전은 Model-centric에서 Data-centric으로 패러다임이 전환되고 있으며, 현재는 양질의 데이터가 모델 아키텍처보다 더 중요한 성공 요소가 되었다.
  • 딥러닝은 Supervised Learning → Semi-Supervised Learning → Self-Supervised Learning 순으로 발전해왔으며, Self-Supervised Learning을 통해 구축된 Foundation Model이 현재 생성형 AI의 핵심이다.
  • Self-Supervised Learning은 데이터 자체를 입력이자 출력으로 사용하여 레이블 없이도 학습이 가능하며, GPT의 '다음 단어 예측' 방식이 대표적인 예시다.
  • 현재 AI 기술의 핵심 경쟁력은 공개된 논문이나 모델이 아닌 실제 개발 과정의 노하우와 데이터 구축 방법론에 있다.
